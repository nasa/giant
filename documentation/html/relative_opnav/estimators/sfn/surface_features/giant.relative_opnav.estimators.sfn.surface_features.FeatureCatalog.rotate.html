FeatureCatalog.rotate¶

giant.relative_opnav.estimators.sfn.surface_features:

FeatureCatalog.rotate(rotation)[source]¶

Rotates the feature catalog.

The rotation is applied to the feature_bounds, feature_normals, and feature_locations. It is not applied to the DEM data for each feature, instead, when rays are traced into the scene they are rotated into the base frame for the feature catalog before tracing. This is generally more efficient because there are normally much fewer rays to trace than DEM points to rotate.
